# ZIP 이미지 도서 뷰어 규칙 관리 문서 (`RULES.MD`)

작성일: 2026-03-06  
목적: 개발 중 생성되는 규칙을 누락 없이 누적/관리하고 즉시 적용하기 위함.

---

## 1) 운영 원칙

- 신규 규칙이 생기면 **당일 즉시** 이 문서에 추가한다.
- 규칙은 반드시 `ID`, `상태`, `적용 범위`, `내용`, `근거`, `생성일`을 포함한다.
- 규칙 충돌 시 우선순위는 `보안 > 데이터 무결성 > 사용자 경험 > 구현 편의` 순으로 판단한다.
- 폐기/변경 규칙은 삭제하지 않고 상태를 `Deprecated`로 변경 후 변경 이력에 남긴다.

## 5) 상태 값 정의

- `Active`: 현재 적용 중
- `Deprecated`: 더 이상 신규 적용하지 않음
- `Superseded`: 다른 규칙으로 대체됨

### R-010 (Active)
- 생성일: 2026-03-07
- 적용 범위: UI / 메뉴 / 파일 진입 구조
- 내용: 기존 좌측 파일탐색기 트리 대신 메뉴 기반 `파일 열기`/`폴더 열기`를 기본 진입으로 사용한다. 폴더 열기 후 좌측에는 폴더 내부의 압축파일/이미지 리스트를 표시하고, 리스트 더블클릭으로 파일을 연다. ZIP은 기존 로직을 사용하고, 이미지 파일은 뷰어에 직접 표시하며, 기타 압축파일은 ZIP 변환 필요 메시지로 처리한다.
- 근거: 탐색기 트리 구조에서 외부 drag hit-test가 불안정했고, 사용자 작업 흐름을 메뉴+리스트 중심으로 단순화하기 위함.
- 관련 파일: PLAN.MD, src/main/electron-main.ts, src/main/ipc/ipc-handlers.ts, src/main/preload.ts, src/renderer/index.ts

### R-011 (Active)
- 생성일: 2026-03-07
- 적용 범위: Viewer / 목록 UI / View 메뉴
- 내용: 좌측 폴더 리스트는 단일 행 축약 목록으로 표시하고 tooltip 으로 전체 파일명을 제공한다. ZIP 뷰어는 1쪽/2쪽 보기를 지원하며, 이미지 단독 보기에는 적용하지 않는다. 이미지 단독 보기에는 자동/1:1/폭맞춤/높이맞춤 표시 옵션을 View 메뉴로 제공한다.
- 근거: 폴더 리스트의 가독성과 뷰어 조작성을 높이고, ZIP/이미지의 보기 정책을 분리하기 위함.
- 관련 파일: PLAN.MD, RULES.MD, src/main/electron-main.ts, src/renderer/index.ts, styles.css

## R-013 앱 상태 영속화
- 앱은 종료 후 재실행 시 마지막 창 상태를 복원해야 한다.
- 최소 저장 항목은 창 위치/크기, 최대화 여부, 언어, 페이지 보기 모드, 이미지 맞춤 모드, 좌측 패널 폭이다.
- 메뉴 체크 상태와 실제 렌더링 상태는 항상 동일해야 한다.
- 이미지 맞춤 "높이에 맞추기"는 세로 기준으로 확대되어 가로 overflow를 허용해야 한다.
- 이미지 맞춤 "폭에 맞추기"는 가로 기준으로 확대되어 세로 overflow를 허용해야 한다.
